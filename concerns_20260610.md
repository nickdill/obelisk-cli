## 1. `.gitignore` substring check lets `.env` slip through (security)

**File:** `cmd/init.go:90`

```go
if !strings.Contains(string(data), ".env") {
```

`strings.Contains` matches any occurrence of the four characters `.env` anywhere in
the file. If an existing `.gitignore` has `.env.local`, `.envrc`, `*.env`, or even a
comment like `# don't commit .env files`, the check passes and the bare `.env` line is
never appended. The generated `.env` file â€” which contains port config today and may
contain secrets in the future â€” is committed to git with no warning.

**Fix:** scan line by line and match the exact token:

```go
for _, line := range strings.Split(string(data), "\n") {
    if strings.TrimSpace(line) == ".env" {
        alreadyIgnored = true
        break
    }
}
```

## 2. Silent `.gitignore` write failure prints false success

**File:** `cmd/init.go:93`

```go
f.WriteString("\n.env\n")   // error discarded
f.Close()                    // error discarded
fmt.Println("  update .gitignore")  // printed unconditionally
```

Both `f.WriteString` and `f.Close` return errors that are never checked. If the write
fails (disk full, read-only filesystem), the user sees `update .gitignore` printed as
if it worked, but the `.env` line was never written.

**Fix:** use `os.WriteFile` with the content already in memory, eliminating the
open-write-close sequence entirely:

```go
updated := append(data, []byte("\n.env\n")...)
if err := os.WriteFile(gitignorePath, updated, 0644); err != nil {
    fmt.Fprintf(os.Stderr, "warning: could not update .gitignore: %v\n", err)
} else {
    fmt.Println("  update .gitignore")
}
```

## 3. Non-purge uninstall silently changed meaning â€” now does almost nothing

**File:** `cmd/uninstall.go:70`

The old non-purge behavior and the new non-purge behavior are very different:

| What is removed | Old `obelisk uninstall` | New `obelisk uninstall` | New `obelisk uninstall --purge` |
|---|---|---|---|
| `.obelisk/` (all scripts) | âś… | âťŚ | âś… |
| `docker-compose.yml` | âś… | âťŚ | âś… |
| `docker-compose.override.yml` | âś… | âś… | âś… |
| `.obelisk/nginx/` | (part of `.obelisk/`) | âś… | (part of `.obelisk/`) |
| `obelisk.yml` | âťŚ (needed `--purge`) | âťŚ | âś… |

In the old design, non-purge was effectively a full uninstall â€” it removed scripts,
docker-compose, and generated files, leaving only the config. `--purge` was the one
extra step that also deleted the config.

The new non-purge only removes generated runtime artifacts (`docker-compose.override.yml`
and the nginx conf dir). The scripts, `docker-compose.yml`, and `.obelisk/` remain on
disk. The success message `"Cleaned generated files. Run 'obelisk dev' to regenerate"`
is consistent with this lighter intent, but represents a semantic shift from the prior
behaviour with no deprecation notice.

**Impact:** A user who ran `obelisk uninstall` before this change to clean up a project
will now find most files still present. If this lighter-clean behaviour is intentional,
the `--purge` flag description needs updating (it currently says "Full teardown: also
remove scripts, docker-compose.yml, and obelisk.yml" implying non-purge removes
everything else â€” but it no longer does).

**Fix options:**
- Restore old behaviour: make non-purge remove `.obelisk/` and `docker-compose.yml`
- Keep new behaviour but update the `--purge` flag description and add explicit messaging
  about what was NOT removed, e.g. `"Scripts left in place. Run 'obelisk uninstall --purge' for full removal."`

## 4. `isServer` ignores `obelisk.local.yml`, rejects valid projects

**File:** `cmd/uninstall.go:36`

```go
isServer := fileExists(filepath.Join(cwd, "obelisk.yml"))
```

`config.Path()` (and therefore `config.Load()`) accepts `obelisk.local.yml` as a valid
server config when `obelisk.yml` is absent. But `isServer` only checks for `obelisk.yml`,
so a project that only has `obelisk.local.yml` on disk â€” a valid and supported setup â€”
hits the `!isModule && !isServer` guard and exits with "no Obelisk project found in
current directory".

**Fix:**

```go
isServer := fileExists(filepath.Join(cwd, "obelisk.yml")) ||
            fileExists(filepath.Join(cwd, "obelisk.local.yml"))
```

## 5. Old-format module projects are misidentified as server projects

**File:** `cmd/uninstall.go:35`

`config.IsModule()` returns true only if `obelisk.module.yml` exists. Before the file
was renamed from `obelisk.yml` (with `type: module`) to `obelisk.module.yml`, any
module project initialized with an earlier version of the CLI will only have `obelisk.yml`
on disk. For those projects:

- `config.IsModule()` â†’ false (no `obelisk.module.yml`)
- `isServer` â†’ true (has `obelisk.yml`)
- The server uninstall path runs, only cleaning generated files
- `obelisk.yml` is left behind and the project is never properly uninstalled

The same issue affects `obelisk status` â€” `config.IsModule()` returns false, so
`showServerStatus` is called with an `obelisk.yml` that has `type: module`, printing
an empty module table instead of module status.

**Fix:** add a migration check: if `obelisk.yml` exists and contains `type: module`,
treat the directory as a module project (and optionally prompt to rename the file).

## 6. `checkGenerateComposeStale` swallows non-`IsNotExist` errors

**File:** `cmd/dev.go:64`

```go
data, err := os.ReadFile(".obelisk/scripts/generate-compose.sh")
if err != nil {
    return nil  // silently skips check for ANY error
}
```

The intent is "if the script doesn't exist yet, skip the check." But any error â€”
including permission denied, I/O failure, or a corrupted file â€” also returns `nil`,
causing `obelisk dev` to proceed silently and run `dev.sh`, which calls the
unreadable script and produces a cryptic shell error instead of the clear
`"run obelisk init --force"` message.

**Fix:**

```go
if err != nil {
    if os.IsNotExist(err) {
        return nil
    }
    return err
}
```
